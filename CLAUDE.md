# CLAUDE.md

Этот файл — правила работы в репозитории. Их читает Claude Code каждого участника артели.
Прочитай его целиком, прежде чем что-либо менять.

## Контекст

Репозиторий проверяет одну гипотезу: смогут ли пять человек договориться про один продукт
(финансовый бот) и получить непротиворечивый набор требований. Подробности — в [README.md](README.md).
Кода здесь пока нет и в ближайшее время не будет. Мы пишем требования, а не приложение.

## Главное правило

**Каждый работает только в своей папке.** Папки участников: `artem/`, `danil/`, `ilya/`,
`vasily/`, `vladimir/`.

Если не знаешь, кто твой пользователь, — **спроси, прежде чем писать хоть один файл**. Не
угадывай по git config: у человека может быть настроен рабочий аккаунт.

Не редактируй:

- чужие папки участников — никогда, даже если видишь там опечатку или противоречие;
- `common/` — туда попадает только результат совместной сборки, по договорённости;
- `README.md` и `CLAUDE.md` — это общие правила, они меняются обсуждением, а не в одиночку.

Правило существует не из вежливости. Оно убирает конфликты слияния и, что важнее, не даёт
нам спорить о требованиях раньше, чем мы их написали.

## Что помочь написать

Основной результат работы участника — один файл: `<папка-участника>/requirements.md`.

Это **не PRD**. Формулировка со встречи: «мы пишем не PRD, мы пишем желание». Человек
рассказывает, чего он хочет от финансового бота, — твоя задача расспросить и оформить.

Три обязательных раздела:

1. **Что я хочу** — какие боли должен закрыть бот.
2. **Чего я точно не хочу** — что раздражает, что не нужно, за что не готов платить.
3. **Что для меня плохо** — при каких условиях продукт лично для этого человека непригоден.

Разговаривай с человеком, а не с формой. Задавай уточняющие вопросы про его реальный опыт:
чем он уже пользовался, что сломалось, откуда берутся его данные (выгрузки из банка, чеки,
Open Banking), где проходит его граница приватности.

## Чего писать не надо

Сознательно вне рамок на этом этапе:

- как это выглядит в интерфейсе, вайрфреймы, макеты;
- пользовательские сценарии по шагам;
- corner-кейсы и обработка ошибок;
- архитектура, стек, выбор библиотек.

Если человек уходит в эти темы — верни его назад. Всё перечисленное только отдалит нас от
ответа на вопрос «договоримся или нет». К UI вернёмся, когда концепция соберётся.

Не раздувай документ. Три честных абзаца полезнее двадцати страниц, которые никто не прочтёт
перед голосованием.

## Этап сборки

Когда все положат свои `requirements.md`, участник может попросить тебя свести их в одну
картину. Тогда:

- прочитай `*/requirements.md` всех участников;
- собери `<папка-участника>/joint-requirements.md` — общий каркас глазами этого участника;
- отдельно и явно выпиши **противоречия**: места, где желания участников несовместимы.
  Не сглаживай их и не выбирай «правильную» сторону. Противоречие — это ценный результат,
  ради него всё и затевалось. Его вынесут на голосование.

Версий `joint-requirements.md` будет столько же, сколько участников. Это ожидаемо. Расхождения
между версиями сами по себе информативны.

## Коммиты

- Коммить только файлы из папки своего участника.
- В `main` изменения приезжают через pull request, даже свои: ветка → PR → merge.
  Для своей папки апрув не нужен — открыл PR и тут же слил сам; проверка `own-folder`
  машинно следит, что автор не тронул чужого. Общие файлы (`README.md`, `CLAUDE.md`,
  `common/`, `.github/`) сливаются только после апрува другого участника — за этим
  следит CODEOWNERS. Как это устроено — в [.github/README.md](.github/README.md).
- Сообщения по-русски, по существу: «требования Ильи: первая версия», а не «update».
- Не делай `git push --force`. Не переписывай историю. Репозиторий публичный.
- Не коммить `.DS_Store`, черновики расшифровок, выгрузки из банков и любые персональные
  финансовые данные. Если человек показывает тебе свою банковскую выгрузку, чтобы объяснить
  формат, — она остаётся у него на диске, в репозиторий попадает только описание формата.

## Приватность

Репозиторий публичный. Всё, что здесь окажется, увидит любой человек в интернете, и удалить
это из форков и веб-архивов будет нельзя.

Расшифровки встреч в репозиторий не входят и не должны в него попасть: там живой разговор
шести человек, которые на публикацию не подписывались. Они лежат выше по дереву, вне
репозитория. Не добавляй их и не цитируй оттуда чужие реплики.

Прежде чем записать в файл чью-то фамилию, место работы, страну проживания или банк, которым
человек пользуется, — спроси, хочет ли он этого.
