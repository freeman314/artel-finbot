## Автоматизация: минимум ручного труда

**Данные приходят из источников сами, ручной ввод — не основной режим.** Ручное
ведение финансов раздражает всех, кто высказался; источник — банки и документы,
а не ежедневная дисциплина в таблице.
— [Василий · 2026-07-10, Илья · 2026-07-13, Владимир · 2026-07-11]

**Realtime не нужен.** Достаточно актуализировать данные раз в неделю-месяц.
— [Владимир · 2026-07-11]

## Источники данных и ввод

**Много банков и счетов из разных стран — в одной картине.** Счета разбросаны по
странам (Испания, Россия, США, СНГ) и валютам; хочу видеть всё в одном месте без
переключения между приложениями банков.
— [Василий · 2026-07-10, Илья · 2026-07-13, Владимир · 2026-07-11]

**Open Banking желателен как источник.** Прямое подключение к банку по API хотят
все трое (обязателен ли он как основной канал — см. противоречие о канале данных).
— [Василий · 2026-07-10, Илья · 2026-07-13, Владимир · 2026-07-11]

**Крипта как источник:** биржи через API, кошельки через сеть (ETH/SOL-трекеры).
Пока вручную (скриншоты → распознавание), хочется автоматизировать.
— [Владимир · 2026-07-11]

**Лог пояснялок для классификации.** Короткие заметки в чат («50 € за клининг»)
как подсказка боту для непонятных переводов; сама запись — не источник финансовых
данных. Отдельно — интеграция с Siri для голосовых пояснялок, особенно наличных.
— [Владимир · 2026-07-11]

**Удобный ввод выписок через Share** (из банка → Share PDF → бот), а не раскладывать
файлы в папку руками.
— [Владимир · 2026-07-11]

## Категоризация

**Свои категории с иерархией.** Иерархия «категория → подкатегория» / «супер → под»,
которую пользователь кроит сам (добавить, переименовать, удалить, свернуть при
анализе). Не навязанный чужой список.
— [Василий · 2026-07-10, Владимир · 2026-07-11]

**Бот учится и не переспрашивает.** Правила классификации бот запоминает; поправку
по мерчанту помнит и больше не ошибается. Приоритет: правки пользователя → общие
справочники → внешние сервисы. Базовый разбор работает без платных ключей.
— [Василий · 2026-07-10, Владимир · 2026-07-11]

**Сырьё сохраняется** — чтобы позже переразбить категории или пере-обработать
старые данные под новую структуру.
— [Владимир · 2026-07-11]

**Сплит одной покупки на несколько категорий** (Amazon-«мега-покупка»: одна
транзакция, внутри книга + бытовое + подарок). В идеале — подтягивать состав
заказа. Помечено автором как **нерешённое желание**, не готовое решение.
— [Владимир · 2026-07-11]

## Мультивалютность

**Итог в одной валюте с сохранением оригинала; два режима.** При анализе всё
сводится в выбранную валюту (у Владимира — евро), при сверке счетов — работа в
исходных валютах. Оригинальная сумма сохраняется, курсы подтягиваются сами.
— [Василий · 2026-07-10, Владимир · 2026-07-11]

## Точность и проверяемость

**Деньги считаются точно, ошибки — ловятся.** Копейки не «плывут» от округлений;
бот не должен тихо ошибаться в цифрах так, что это не поймать. Проверяемость —
через снимки счетов, контрольные суммы, sanity-метрики (ловить «гонево»: обмен,
которого не было; «клининг за 5000 €»).
— [Василий · 2026-07-10, Владимир · 2026-07-11]

## Аналитика и доступ

**Поиск любой траты.** По мерчанту, фильтры по дате, категории, сумме, типу
(расход/доход).
— [Василий · 2026-07-10]

**Аналитика, а не просто список.** Помесячно доход/расход, разбивка по категориям,
топ мерчантов; клик по категории на графике → провалиться в её транзакции.
— [Василий · 2026-07-10]

**Дашборд + доступ через своих агентов (MCP-слой).** Обращаться к боту как к
финаналитику: что потратил, где неочевидные расходы, что проверить — краткая сводка.
— [Василий · 2026-07-10]

**Предсказание ближайших трат** (аренда, кредиты, поездки, налоги, регулярные
обязательства). Источник — авто-распознавание повторяющихся платежей + ручное
добавление того, что бот не увидел.
— [Илья · 2026-07-13]

**Пуши в двух случаях:** (1) бот не смог классифицировать перевод и спрашивает
«что это?»; (2) давно не вводил данные — пора. Отдельного приложения не требуется.
— [Владимир · 2026-07-11]

## Хранение данных

**Данные принадлежат пользователю, без вендор-локина.** Хранятся в своём хранилище
(Dropbox / свой сервер / NAS), забрать всё накопленное можно в любой момент. Без
зависимости от чужого постоянного бэкенда.
— [Василий · 2026-07-10, Илья · 2026-07-13, Владимир · 2026-07-11]

_(Насколько строго «локально по умолчанию» и допустимо ли облако для обработки —
см. противоречие о приватности.)_

## Предпринимательский / налоговый слой

**Цифровой gestor для autónomo (Испания) и ИП (РФ).** Собирает материал по
предпринимательской деятельности, помогает готовить отчётность и налоги. Не
заменяет живого бухгалтера, снимает рутину сбора данных.
— [Илья · 2026-07-13]

## Персонализация и форк

**Форкнуть и допилить под себя** — свой банк, свои категории, свой вид — не
переписывая продукт целиком. Общий фундамент, сверху тонкий слой под каждого.
— [Василий · 2026-07-10]

**Свой скин поверх общих сценариев.** Настраивать оформление (тема, цвета, скин),
но пользовательские сценарии (flow) — общие для артели, обсуждаются вместе.
Персонализация — слой оформления, а не пять разных продуктов.
— [Василий · 2026-07-11]

**Совместность с партнёром.** Общий бюджет с женой; в формате бота в общем канале
решается само; если появится приложение — связать два в один аккаунт.
— [Владимир · 2026-07-11]

## Связка чеков и транзакций

**Чек ↔ транзакция.** Если чек оплачен картой и карта есть в транзакциях — связать
чек с конкретной транзакцией, показать разбивку по позициям. Полезно для поиска:
начать с чека и дойти до транзакции.
— [Василий · 2026-07-10]
